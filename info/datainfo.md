# 資料欄位說明文件

## 1. 交易資料 (acct_transaction.csv)

交易資料包含玉山銀行帳戶間的轉帳交易記錄。

| 欄位名稱 | 欄位中文名稱 | 欄位中文說明 | 資料格式 | 備註 |
|---------|-------------|------------|---------|------|
| from_acct | 轉出帳號 | 轉出交易的帳號 | 類別型 | - |
| from_acct_type | 轉出帳號是否為玉山帳號 | 標記轉出帳號的所屬機構 | 類別型 | 01: 玉山<br>02：其他金融機構 |
| to_acct | 轉入帳號 | 接收交易的帳號 | 類別型 | - |
| to_acct_type | 轉入帳號是否為玉山帳號 | 標記轉入帳號的所屬機構 | 類別型 | 01: 玉山<br>02：其他金融機構 |
| is_self_txn | 交易是否為同一人 | 判斷轉出與轉入是否為同一人 | 類別型 | Y: 轉出帳號和轉入帳號屬於同一人<br>N: 轉出帳號和轉入帳號屬於不同人<br>UNK: 無法判定或空值 |
| txn_amt | 交易金額 | 本次交易的金額 | 數值型 | 交易金額皆為正值，經過轉換存取 |
| txn_date | 交易日期 | 交易發生的日期 | 類別型 | 競賽第一日為1，以此類推 |
| txn_time | 交易時間 | 交易發生的時間 | 數值型 | - |
| currency_type | 幣別 | 交易使用的貨幣類型 | 類別型 | 可能超過10種幣別<br>TWD：新台幣<br>USD：美金 |
| channel_type | 交易通路 | 交易發生的通路管道 | 類別型 | 參閱下方通路代碼表 |

### 交易通路代碼表

| 代碼 | 通路名稱 |
|------|---------|
| 1 | ATM |
| 2 | 臨櫃 |
| 3 | 網路銀行(網路) |
| 4 | 行動銀行(手機) |
| 5 | 約定 |
| 6 | eATM |
| 7 | 電子支付 |
| 99 | 系統批次作業 |
| UNK | 無法識別或空值 |

## 2. 警示帳號清單 (acct_alert.csv)

警示帳號清單包含已知的警示帳號資訊。

| 欄位名稱 | 欄位中文名稱 | 欄位中文說明 | 資料格式 | 備註 |
|---------|-------------|------------|---------|------|
| acct | 玉山帳號 | 帳戶識別碼 | 類別型 | 此檔案的帳號警示標記(label)皆為1<br>1：為警示帳號<br>0：非警示帳號 |
| event_date | 警示日期 | 帳號被列為警示的日期 | 類別型 | 競賽第一日為1，以此類推 |

## 3. 預測清單 (acct_predict.csv)

預測清單包含需要預測的玉山帳號。

| 欄位名稱 | 欄位中文名稱 | 欄位中文說明 | 資料格式 | 備註 |
|---------|-------------|------------|---------|------|
| acct | 玉山帳號 | 需要預測的帳戶識別碼 | 類別型 | 會於競賽提供 public & private |
| label | 預測結果 | 預測該帳號是否為警示帳號 | 類別型 | 1：預測為警示帳號<br>0：預測非警示帳號 |

## 資料處理注意事項

1. **日期處理**：所有日期欄位皆以競賽第一日為1開始計算，為相對日期格式
2. **金額處理**：交易金額已經過轉換，皆為正值
3. **缺失值處理**：部分欄位可能包含 UNK 值，表示無法識別或空值

## 資料使用建議

1. **時序分析**：可利用 txn_date 和 txn_time 進行時間序列分析
2. **網路分析**：可利用 from_acct 和 to_acct 建立交易網路圖
3. **特徵工程**：
   - 計算帳戶的交易頻率、金額統計等特徵
   - 分析不同交易通路的使用模式
   - 識別異常交易行為模式
4. **標籤使用**：acct_alert.csv 提供的警示帳號可作為訓練標籤使用
5. **預測比例** :預測的資料有 4,781 筆玉山帳號 有 240 筆為 1

# 核心挑戰

1. **極度不平衡**
   - Y=1（警示帳戶）：1,004 個
   - 候選負樣本池：1,800,106 個帳戶
   - 不平衡比例：1:1,793

2. **分布差異（Domain Shift）**

| 特徵 | Y=1（警示帳戶）| df_acct_pred（預測目標）| 倍數差異 |
|------|---------------|----------------------|----------|
| txn_count（平均）| 33.88 | 77.78 | **2.3x** |
| txn_amt_sum（平均）| 643,268 | 3,632,407 | **5.6x** |
| txn_amt_mean（平均）| 28,047 | 50,943 | **1.8x** |
| txn_amt_std（平均）| 29,561 | 66,496 | **2.2x** |

**關鍵發現**：預測目標的帳戶活躍度遠高於已知警示帳戶，這是嚴重的 domain shift 問題。

3. **負樣本標註不確定性**
   - 180 萬個候選負樣本中，可能包含未被偵測的警示帳戶
   - 負樣本 ≠ 絕對安全帳戶，只是「未被標記」
